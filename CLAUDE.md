# 소리튠 주니어 — 프로젝트 가이드

## 0. 개발/배포 워크플로우

| 환경 | 디렉토리 | 브랜치 |
|------|----------|--------|
| 개발 (JUNIOR) | `/var/www/html/_______site_SORITUNECOM_JUNIOR/` | `dev` |
| 운영 (J) | `/var/www/html/_______site_SORITUNECOM_J/` | `main` |

- 코드 작업은 항상 **JUNIOR(dev)** 에서만 진행한다.
- 운영 반영은 사용자가 요청할 때만: dev → main 머지 + J 디렉토리에서 `git pull`.
- J(운영)에 직접 커밋/푸시하지 않는다.

# Voice & Tone Guide (v2)

앱 전체 UX writing에 적용하는 톤 가이드.

---

## 1. 대상 & 페르소나

| 구분 | 설명 |
|------|------|
| 주 사용자 | 초등 1~3학년 (7~9세), 영어 초급~중급 |
| 보조 사용자 | 학부모, 코치/선생님 |
| 앱 페르소나 | "같이 놀아주는 친한 형/누나" — 선생님 ✗, 친구 ✗ (너무 가벼움), 코치 ✗ (너무 무거움) |
| 관계 톤 | 응원하되 과잉 칭찬 안 함. 실패를 언급 안 함. 다음 행동을 알려줌. |

## 2. 핵심 톤 원칙 (5가지)

### 🟢 DO — 항상 지킨다

| # | 원칙 | 설명 | 예시 |
|---|------|------|------|
| 1 | 반말 + 짧은 문장 | 존댓말 절대 ✗. 한 문장 15자 이내 권장. 최대 20자. | 읽어봐! 골라봐! 다시 해볼까? |
| 2 | 행동 유도형 | "~입니다" 상태 설명 ✗. "~해봐" 다음 행동 ✅ | ✗ "녹음이 완료되었습니다" → ✅ 다 읽었어! |
| 3 | 긍정 프레이밍 | 실패/오류도 긍정적으로. 아이 탓 절대 ✗ | ✗ "틀렸어" → ✅ 선택 완료! |
| 4 | 이모지 = 기능 아이콘 | 장식 ✗. 의미 구분자로만 사용. 문장당 1개 | 🎤 눌러서 읽어봐! 💰 코인 3개 받았어! |
| 5 | 정보는 숫자로, 감정은 이모지로 | 점수/순위 → 숫자. 느낌 → 이모지 | 🏅 3등! 😊 쉬웠어! |

### 🔴 DON'T — 절대 하지 않는다

| 금지 | 이유 | 잘못된 예 |
|------|------|-----------|
| 존댓말 (~합니다, ~하세요, ~입니다) | 딱딱함, 선생님 느낌 | ✗ "녹음해 주세요" |
| 정답/오답 즉시 피드백 (테스트 시) | 시험 느낌, 위축됨 | ✗ ❌ 틀렸어! ✅ 정답! |
| 점수/퍼센트를 아이에게 노출 (테스트 시) | 비교/경쟁 유발 | ✗ "80점이야" "3/5 맞았어" |
| 긴 설명문 (3줄 이상) | 아이는 안 읽음 | ✗ "이 단계에서는 파닉스를 통해..." |
| 느낌표 2개 이상 | 과잉 흥분 | ✗ 대단해!! 최고야!!! |
| 과잉 칭찬 (천재, 최고, 완벽) | 부담, 신뢰 하락 | ✗ "넌 천재야!" |
| 부정어 시작 (못, 안, 없) | 위축됨 | ✗ "코인이 없어" → ✅ 코인을 모아보자! |
| 비교 표현 (다른 친구보다, 평균보다) | 경쟁 유발 | ✗ "반 평균보다 높아!" |

## 3. 문맥별 카피 패턴

### 3-1. 🎓 테스트 / 학습 화면

> 원칙: 시험이 아닌 놀이. 정답/오답 안 보여줌.

| 상황 | 패턴 | 예시 |
|------|------|------|
| 지시문 | [이모지] + ~해봐/~어볼까 | 이 글자를 소리내어 읽어봐! |
| 단어 뜻 | 이 단어의 뜻은? | 물음표로 끝남 (명령 ✗) |
| 블록 힌트 | 💡 + 한국어 해석 | 💡 이런 뜻의 문장을 만들어봐! |
| 녹음 시작 | 🎤 눌러서 읽어봐! | |
| 녹음 중 | 다 읽었으면 ■ 눌러! | |
| 자기평가 | 감정 3단계 (아래 참조) | |
| 답 선택 후 | 선택 완료! | 맞았는지 안 알려줌 |
| 다음 스텝 | 다음으로! ✨ | |
| 포기 | 너무 어려워요 😵 | 아이가 스스로 누르는 버튼 |

**자기평가 버튼 (녹음 후 고정)**

| 색 | 라벨 | 값 |
|----|------|----|
| 초록 | 😊 쉬웠어! | easy |
| 노랑 | 🤔 좀 어려웠어 | okay |
| 빨강 | 😵 어려웠어 | hard |

### 3-2. 💰 코인 / 보상

> 원칙: 얻을 때 기뻐하고, 부족할 때 격려. 잃는 느낌 ✗.

| 상황 | 카피 |
|------|------|
| 코인 획득 | 💰 코인 3개 받았어! |
| 코인 잔액 | 💰 12 (숫자만, 설명 ✗) |
| 코인 사용 | 💰 코인 5개 사용! |
| 코인 부족 | 코인이 조금 부족해. 도전해서 더 모아보자! |
| 보너스 코인 | 🎁 보너스 코인이다! |
| 연속 출석 보상 | 🔥 3일 연속! 코인 2배! |

### 3-3. 🏅 랭킹 / 순위 / 리더보드

> 원칙: 등수는 보여주되, 비교/경쟁 표현 ✗. 본인 성장에 초점. "~보다" 절대 ✗.

| 상황 | 카피 |
|------|------|
| 내 순위 표시 | 🏅 이번 주 {n}등! |
| 순위 상승 | ⬆️ 2칸 올랐어! |
| 순위 유지 | 💪 자리 지키는 중! |
| 순위 하락 | (표시 안 함. 하락은 언급 ✗) |
| 1등 | 🥇 1등이야! |
| 랭킹 페이지 타이틀 | 이번 주 랭킹 |
| 빈 랭킹 | 아직 도전한 친구가 없어. 첫 번째가 되어볼까? |

### 3-4. 🏆 업적 / 뱃지 / 레벨업

> 원칙: 과정을 인정. "잘했어"보다 "해냈어" 사용.

| 상황 | 카피 |
|------|------|
| 뱃지 획득 | 🏆 [뱃지명] 뱃지 획득! |
| 레벨업 | ⭐ Level {n} 달성! |
| 연속 학습 | 🔥 {n}일 연속 도전 중! |
| 연속 끊김 | (언급 ✗. 다음 접속 시) 다시 시작해보자! |
| 목표 달성 | 🎯 목표 달성! 대단해! |
| 목표 미달 | 조금만 더! 내일 다시 해보자. |
| 전체 완료 | 모든 레벨 클리어! 🚀 |

### 3-5. ⚠️ 오류 / 에러 메시지

> 원칙: 아이 탓 ✗. 원인은 간단히, 해결 행동 제시. 기술 용어 절대 ✗ (서버, 네트워크, 타임아웃 등).

| 상황 | 카피 |
|------|------|
| 인터넷 끊김 | 📡 인터넷이 잠깐 쉬고 있어. 다시 연결해줘! |
| 서버 오류 | 🔧 잠깐 문제가 생겼어. 조금만 기다려줘! |
| 로딩 실패 | 다시 불러올게. 잠깐만! |
| 마이크 권한 없음 | 🎤 마이크를 켜줘야 읽을 수 있어! |
| 마이크 인식 실패 | 🎤 잘 안 들렸어. 더 크게 읽어볼까? |
| 파일 로드 실패 | 📂 불러오지 못했어. 다시 해볼까? |
| 타임아웃 | ⏰ 시간이 좀 걸리고 있어. 잠깐만 기다려! |
| 알 수 없는 오류 | 앗, 뭔가 이상해! 다시 해보자. |
| 업데이트 필요 | 🆕 새 버전이 나왔어! 업데이트하자. |
| 브라우저 미지원 | 이 브라우저에서는 안 돼. 크롬을 써줘! |

### 3-6. 📭 빈 상태 (Empty State)

> 원칙: 비어있는 게 나쁜 것 ✗. 채울 수 있다는 행동 유도.

| 상황 | 카피 |
|------|------|
| 학습 기록 없음 | 아직 기록이 없어. 첫 도전 해볼까? |
| 코인 0개 | 코인을 모아보자! 도전하면 받을 수 있어. |
| 뱃지 없음 | 뱃지가 기다리고 있어! 도전해보자. |
| 알림 없음 | 새 소식이 없어. 조용한 하루! |
| 검색 결과 없음 | 찾는 게 없어. 다른 걸로 해볼까? |
| 친구 목록 없음 | 아직 친구가 없어. 초대해볼까? |

### 3-7. 🔄 로딩 / 대기

> 원칙: 짧고 가볍게. 기술 설명 ✗.

| 상황 | 카피 |
|------|------|
| 일반 로딩 | 잠깐만... |
| 데이터 불러오기 | 가져오는 중... |
| 저장 중 | 저장 중... |
| 채점 중 (내부) | 확인하는 중... |
| 긴 로딩 (5초+) | 거의 다 됐어! |

### 3-8. 🧭 네비게이션 / 버튼 라벨

> 원칙: 2~4글자 동사형. 명사형 ✗.

| 상황 | 카피 | 잘못된 예 |
|------|------|-----------|
| 다음으로 | 다음 → | ✗ "다음 페이지" |
| 이전으로 | ← 이전 | ✗ "뒤로 가기" |
| 시작 | 시작하기! | ✗ "테스트 시작" |
| 다시 하기 | 🔄 다시 하기 | ✗ "재시도" |
| 홈으로 | 🏠 홈 | ✗ "메인으로 돌아가기" |
| 설정 | ⚙️ 설정 | |
| 닫기 | 닫기 | ✗ "창 닫기" |
| 확인 | 알겠어! | ✗ "확인" (딱딱함) |
| 취소 | 안 할래 | ✗ "취소" |
| 저장 | 저장! | |
| 공유 | 공유하기 | |
| 삭제 확인 | 정말 지울까? | ✗ "삭제하시겠습니까?" |

### 3-9. 🔔 알림 / 푸시

> 원칙: 한 줄. 행동 유도. 긴급함 ✗.

| 상황 | 카피 |
|------|------|
| 학습 리마인더 | 오늘 아직 안 했지? 5분이면 돼! |
| 새 레벨 열림 | 🆕 새 레벨이 열렸어! |
| 연속 학습 위기 | 🔥 연속 기록 지키러 가자! |
| 친구 초대됨 | 👋 새 친구가 왔어! |
| 주간 리포트 | 📊 이번 주 결과가 나왔어! |

### 3-10. 👨‍👩‍👧 학부모/선생님 전용 영역

> 원칙: 아이 톤과 분리. 존댓말 OK. 데이터 중심. 단, 앱 전체 톤과 너무 동떨어지지 않게.

| 상황 | 톤 | 예시 |
|------|-----|------|
| 결과 리포트 | 존댓말, 객관적 | 이번 주 학습 결과입니다. |
| 코치 코멘트 | 반말 유지 (아이에게 말하는 톤) | {name}! 파닉스부터 차근차근 하자! 🌱 |
| 선생님 라벨 | 작은 글씨 | (선생님용) (상세 데이터) |
| 학습 통계 | 숫자 + 간단 해석 | 정답률 72% · 문장 읽기가 강해요 |

> **핵심 구분:** 아이에게 보이는 곳 = 반말 + 감정. 선생님만 보는 곳 = 존댓말 + 데이터.

## 6. Claude Code 사용법

### 새 파일/화면 만들 때
```
CLAUDE.md 톤으로 [화면 설명]을 만들어줘.
```

### 기존 파일 톤 수정
```
CLAUDE.md 읽고 [파일명]의 모든 한국어 UX 텍스트를
가이드에 맞게 수정해줘. 변경 전/후를 표로 보여줘.
```

### 톤 검수 (위반 찾기)
```
CLAUDE.md 기준으로 [파일명]을 검수해줘.
위반 항목만 알려줘.
```

### 특정 상황 카피 요청
```
CLAUDE.md 톤으로 "코인 부족 모달" 카피를 써줘.
```

## 7. 체크리스트

새 카피를 쓸 때 아래를 확인한다:

- [ ] 반말인가? (~해봐, ~야, ~어!)
- [ ] 15자 이내인가? (최대 20자)
- [ ] 행동을 유도하는가? (상태 설명 ✗)
- [ ] 정답/오답을 아이에게 드러내지 않는가? (테스트 시)
- [ ] 점수/퍼센트가 아이에게 안 보이는가? (테스트 시)
- [ ] 이모지가 기능적으로 쓰였는가? (장식 ✗)
- [ ] 느낌표가 문장당 1개 이하인가?
- [ ] 부정어로 시작하지 않는가? (못, 안, 없)
- [ ] 비교 표현이 없는가? (~보다, 평균)
- [ ] 기술 용어가 없는가? (서버, 네트워크, 에러)
